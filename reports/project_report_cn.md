# IMDB 电影数据分析与票房预测项目报告

## 1. 引言

在电影产业中，票房表现一直是衡量影片商业成功与否的核心指标之一。无论是制片公司、发行公司、流媒体平台，还是内容研究人员，都希望回答几个非常实际的问题：什么类型的电影更容易取得商业成功，不同年份的电影市场发生了怎样的变化，预算、类型、语言、上映时间等结构化信息是否能够解释部分票房差异，以及在缺少更复杂行业变量的前提下，历史电影元数据能否支持一个具有一定解释力的票房预测模型。

与许多只停留在静态统计图表的课程项目不同，本项目尝试把“数据准备、分析解释、可视化表达、预测验证和交互展示”组织成一个完整流程。项目基于 Kaggle 的 The Movies Dataset 及 IMDb 相关元数据，构建了统一的数据清洗管道、探索性数据分析模块、高级分析模块、票房预测模块和 Tkinter 图形界面，使清洗后的同一份标准化数据能够同时服务于命令行报告、图表导出、预测回测和交互式浏览。

因此，本项目的目标不仅是描述电影数据本身，更是通过一个相对完整、可复现、可展示的工程化分析流程，展示数据分析如何从一个实际问题出发，逐步形成结构化结论：电影市场的整体结构如何，不同类型的质量与商业表现有何差异，电影工业在长期时间维度上呈现出怎样的趋势，以及基于结构化历史特征的票房预测在多年度回测中能够达到什么程度的表现。

## 2. 项目目标与研究问题

本项目围绕以下四类目标展开：

1. 构建统一、稳定、可重复使用的电影数据清洗与预处理流程。
2. 通过 EDA 和图表分析识别电影市场中的分布特征、时间趋势和变量关系。
3. 基于历史结构化元数据建立票房预测模型，并通过滚动回测进行验证。
4. 将分析结果封装为可视化产物和交互式应用，提高项目展示性和可用性。

围绕这些目标，项目主要关注以下研究问题：

- 电影数据中最常见的类型是什么，它们在评分与票房表现上有何差异？
- 电影产量、评分和票房是否存在显著的长期时间趋势？
- 预算与票房之间是否存在稳定的相关关系？
- 基于预算、片长、上映时间、语言、国家和类型等结构化特征，是否能够对票房形成一定程度的预测？
- 若从工程实现角度看，如何将数据清洗、分析、图表、预测和 UI 统一到一套项目结构中？

## 3. 数据来源与研究范围

### 3.1 数据来源

项目原始数据来自 Kaggle 的 The Movies Dataset，并主要使用以下表：

- `movies_metadata.csv`
- `keywords.csv`
- `credits.csv`
- `ratings_small.csv`
- `links_small.csv`

其中，`movies_metadata.csv` 提供影片标题、发布日期、预算、票房、评分、时长、语言等核心元信息；`keywords.csv` 提供关键词信息；`credits.csv` 提供演职员相关信息。项目在原始阶段对这些表进行合并，并在后续统一映射为适合分析的字段结构。

### 3.2 研究时间范围

为保证项目分析口径一致，清洗阶段将所有 `2017-07-31` 之后的影片记录剔除。因此，当前项目研究范围覆盖：

- 起始日期：`1874-12-09`
- 结束日期：`2017-07-29`
- 数据截断日期：`2017-07-31`

这意味着项目中的 `2017` 年并不是完整自然年，而是仅覆盖 `2017-01` 到 `2017-07` 的数据。因此，在解释图表与预测结果时，`2017` 应理解为 `2017 (Jan-Jul only)`。

### 3.3 清洗后数据规模

根据当前项目运行结果，清洗后的标准化数据集包含：

- 样本量：45,293 部电影
- 主类型数量：21
- 平均评分：5.63
- 时长中位数：95 分钟
- 总票房：1,194,684,944,135 美元

这一规模足以支持长期趋势分析、类型比较分析、预算与票房关系分析，以及多年滚动票房回测。

## 4. 项目整体架构

项目并不是单文件分析脚本，而是以模块化方式组织为一个完整应用。主要结构包括：

- `data_preprocessing.py`
  负责数据读取、原始表合并、数据清洗、字段标准化、质量报告生成和缓存管理。
- `data_analysis.py`
  负责筛选逻辑、类型聚合、年度聚合、Top Movies 排序以及自动洞察生成。
- `advanced_analytics.py`
  负责时间序列摘要、预算与票房相关性、年代比较以及高票房特征画像。
- `data_visualization.py`
  负责生成项目使用的图表，并带有回测结果缓存与图表导出逻辑。
- `box_office_forecasting.py`
  负责票房预测数据准备、特征构建、模型训练、年度验证、多年回测和结果导出。
- `imdb_ui.py`
  提供 Tkinter 图形界面，用于筛选、浏览、查看洞察和预览图表。
- `main.py`
  统一入口，支持 `report` 与 `ui` 两种运行模式。

从工作流角度看，项目可概括为：

`原始电影记录 -> 清洗后的统一数据集 -> EDA 与高级分析 -> 图表导出 -> 票房预测回测 -> UI 交互展示`

这种架构的优点在于：不同模块共享同一份清洗后数据，因此分析口径一致；同时由于清洗结果可缓存，命令行和 UI 启动速度更快，维护成本也更低。

## 5. 数据清洗与预处理设计

### 5.1 原始数据合并策略

项目首先加载原始 Kaggle 表，并以 `movies_metadata.csv` 为主表。随后通过电影 `id` 将关键词和演职员信息合并到主表中。合并过程中会首先将 `id` 转换为数值类型，并删除无法解析的非法标识。

在字段映射上，项目会把多个潜在来源合并为统一字段，例如：

- 主类型统一为 `primary_genre`
- 语言统一为 `language`
- 国家统一为 `country`
- 关键词统一为 `keyword`

这样做可以减少后续分析模块处理原始复杂字段结构的负担。

### 5.2 日期清洗与研究窗口控制

项目对 `release_date` 进行严格标准化，无法转换为合法日期的记录将被删除。之后，所有晚于 `2017-07-31` 的记录都会被移除，并自动生成 `year` 字段。

这一处理有两个作用：

- 保证时间分析和回测范围一致。
- 避免使用不完整或超出数据可靠窗口的记录。

### 5.3 重复值处理与标题歧义消解

项目的去重逻辑并非简单按标题去重，而是引入了更谨慎的多级处理：

- 优先按原始 `id` 去重。
- 对标题、年份、原始标题和语言构成的签名重复进行二次去重。
- 对同名同年份但实际并非同一部电影的记录进行标题消歧。

为了提升保留记录的质量，项目为每一行计算质量分数，优先保留信息更完整、观测字段更多的记录。对于确实存在标题冲突的情况，系统会尝试使用原始标题、完整日期、语言或原始标识生成更具区分性的标题。

这种设计优于简单粗暴的标题去重，因为它尽量减少误删不同电影的风险。

### 5.4 缺失值与数值字段处理

项目对核心数值字段进行了系统整理，包括：

- `budget`
- `revenue`
- `runtime`
- `vote_average`
- `vote_count`
- `popularity`

处理逻辑主要包括：

- 非法字符串转为数值时使用容错转换。
- 对 `budget`、`revenue`、`runtime` 等字段，将非正值视为缺失。
- 对部分数值列优先按类型中位数填补，再使用全局中位数或默认值兜底。
- 对 `vote_average` 做 `0-10` 区间裁剪。

此外，项目还引入了三个非常关键的观测标记字段：

- `budget_observed`
- `revenue_observed`
- `runtime_observed`

这些字段用于区分“原始数据中真实存在的值”和“清洗阶段填补得到的值”，尤其对预测模块非常重要，因为模型需要知道某个预算或时长是否来自真实观测，而不是后期补齐。

### 5.5 文本字段标准化

文本字段中常见的空字符串、空白字符、不规范分隔符等问题会被统一处理。对于关键文本字段，如：

- `primary_genre`
- `country`
- `keyword`
- `language`

若清洗后仍缺失，系统会使用 `Unknown` 或可追溯来源信息进行填补，保证清洗后的最终数据表不存在空值。

### 5.6 清洗后字段结构

当前项目清洗后的核心字段包括：

其中：

- `source_id` 是原始数据中的电影标识，便于追踪原始来源。

## 6. 描述性分析方法

### 6.1 数据筛选机制

项目支持按以下条件对数据进行筛选：

- 类型
- 起始年份
- 结束年份
- 最低评分
- 语言
- 国家
- 标题/关键词搜索

其中，标题与关键词搜索采用“包含即命中”的方式：

- 标题中包含输入关键词的电影会被保留
- 关键词字段中包含输入关键词的电影也会被保留
- 两类结果会合并展示，确保用户可以看到所有匹配电影

这使得 UI 搜索结果更完整，也更符合用户对搜索功能的直觉预期。

### 6.2 类型聚合分析

`aggregate_by_genre` 会将 `primary_genre` 展开后按类型汇总，生成：

- `movie_count`
- `avg_rating`
- `avg_revenue`

该结果用于类型分布图和类型比较图，也为自动洞察提供输入。

### 6.3 年度聚合分析

`yearly_rating_trend` 会按年份聚合生成：

- `movie_count`
- `avg_rating`
- `avg_revenue`

这为时间趋势分析和图表生成提供了基础。

### 6.4 自动洞察生成

项目并不是只输出数据表，还会自动生成简短、适合汇报的英文洞察，例如：

- 当前样本中数量最多的类型
- 可比较类型中平均评分最高的类型
- 产量最高的年份
- 平均评分最高的年份
- 预算与票房的相关关系

这提升了项目的演示友好度和汇报效率。

## 8. 高级分析方法

除了基础 EDA 之外，项目还加入了若干高级分析模块，用于补充更高层次的结论。

### 8.1 年度产量趋势分析

`analyze_yearly_production` 会按年份统计电影数量和平均评分，并使用线性拟合估计长期趋势方向与斜率。输出包括：

- 研究时间段
- 总电影数
- 年均产量
- 产量峰值年份
- 趋势方向
- 趋势斜率
- 平均增长率

### 8.2 时间序列摘要分析

`analyze_time_series_summary` 在年度聚合基础上进一步构建：

- 电影数量三年移动平均
- 票房三年移动平均
- 电影数量同比增速
- 票房同比增速
- 电影数量与票房的长期趋势斜率

这使时间分析不再局限于“看图说话”，而是能够用结构化指标表达变化特征。

### 8.3 预算与票房回归分析

`analyze_budget_revenue_correlation` 将预算与票房视为回归变量，输出：

- 样本量
- 相关系数
- 相关强度描述
- 线性回归斜率和截距
- `R^2`
- 平均 ROI
- 中位 ROI
- 最佳 ROI 影片

### 8.4 年代比较分析

`decade_comparison` 按年代聚合统计：

- 电影数量
- 平均评分
- 平均时长
- 总票房

这有助于从更长时间尺度讨论电影工业的演变。

### 8.5 高票房电影特征画像

`profile_high_box_office_characteristics` 是项目中较有特色的高级模块。它使用一个基于对数票房的岭回归模型，对高票房电影的结构化特征进行总结，并输出：

- 高票房阈值
- 评估集上的 `R^2`、MAE、WAPE
- 正向影响较大的特征
- 负向影响较大的特征
- 预测为高票房影片的典型画像

这一模块本质上是将“预测模型”反向用于“可解释分析”。

## 9. 票房预测模块设计

### 9.1 问题定义

票房预测模块的目标不是对单部电影做工业级商业预测，而是在课程项目范围内验证一个更基础的问题：

“如果只使用电影的结构化历史元数据，是否能够对票房形成一定程度的解释和预测？”

因此，项目更强调：

- 时间顺序上的合理验证
- 特征设计的可解释性
- 多模型比较
- 年度总量层面的趋势跟踪

### 9.2 预测数据准备

预测模块首先对清洗后数据进行二次准备：

- 仅保留具有合法发布日期的记录
- 删除重复行
- 统一主类型、语言和国家字段
- 生成 `year` 和 `month`
- 对 `budget`、`revenue`、`runtime` 转为数值型
- 使用 `*_observed` 标识保留真实观测信息
- 对无效或非正值目标票房保留缺失，不进行目标值填补

这一点非常重要：预测模块不会像描述性分析那样对目标 `revenue` 做无条件补全，而是尽可能保留真实缺失状态，避免对模型训练产生污染。

### 9.3 特征工程

当前项目使用的主要特征包括：

- `log_budget`
- `log_budget_sq`
- `runtime`
- `runtime_sq`
- `year`
- `month_sin`
- `month_cos`
- `is_english`
- `is_us_production`
- `budget_observed`
- `runtime_observed`

此外，对于线性类模型，项目还会加入：

- 类型哑变量
- `log_budget` 与类型的交互项

其中：

- `month_sin` 和 `month_cos` 通过周期方式编码月份，避免将月份误当作线性数值。
- 预算和时长二次项用于捕捉一定的非线性关系。
- `is_english` 和 `is_us_production` 用于反映英语电影与美国制作电影在全球票房市场中的结构性优势。

### 9.4 模型类型

当前项目支持四类模型：

- `ridge`
  岭回归，也是默认模型。
- `ols`
  普通最小二乘回归。
- `genre_budget_ratio`
  按类型估计票房/预算比的简化基线模型。
- `random_forest`
  随机森林回归模型。

默认参数包括：

- 默认验证年份：`2017`
- 岭回归正则化系数 `alpha = 8.0`
- 最小训练样本数：`12`
- 随机森林树数：`300`
- 随机森林最小叶节点样本数：`3`

### 9.5 校准机制

项目的一个重要设计是：模型预测并不是直接输出，而是经过两层校准。

第一层是全局线性校准：

- 根据训练集上的预测值和真实值拟合校准斜率和截距。

第二层是类型级校准：

- 若某一类型训练样本足够，则对该类型单独拟合校准。
- 若类型样本不足，则在全局校准和类型缩放之间进行混合。

这种设计的目的是缓解不同类型在票房规模上的系统性偏差，使同一个全局模型在类型层面更稳定。

### 9.6 验证方式

项目采用滚动留出验证而非随机划分。对于每一个验证年份：

- 训练集仅使用早于该年的历史数据。
- 验证集仅使用该年份的数据。

这种方法比随机 train-test split 更符合真实预测情境，因为它避免了未来信息泄露。

### 9.7 评估指标

项目在类型级和整体级均计算以下指标：

- MAE
- RMSE
- MAPE
- WAPE
- `R^2`

其中：

- WAPE 对总量误差更直观，适合比较不同年份的整体偏差。
- `R^2` 用于衡量拟合解释度。
- MAPE 在极小票房影片上会被放大，因此在报告中不应单独作为唯一判断依据。

## 10. 项目主要结果

## 12. 多年度滚动回测结果（2003-2017）

为了避免只看单一年份，本项目进一步对 `2003-2017` 共 15 个验证年份进行逐年滚动回测。每一年都严格使用此前历史数据训练，再对当年数据进行验证。

### 12.1 整体回测汇总

当前默认岭回归模型的整体回测结果为：

- 回测年份数：15
- 平均 WAPE：59.98%
- 中位 WAPE：58.61%
- 平均 `R^2`：0.6057
- 最佳年份：2010，WAPE 53.60%
- 最差年份：2006，WAPE 67.14%

这说明模型并不是只在某一个验证年份上偶然有效，而是在 15 个年份中都保持了中等水平的稳定解释力。

### 12.2 各年度结果特征

从详细结果可以看到：

- `2003`：WAPE 56.82%，`R^2 = 0.6747`
- `2004`：WAPE 66.67%，`R^2 = 0.4160`
- `2008`：WAPE 64.07%，`R^2 = 0.5275`
- `2010`：WAPE 53.60%，`R^2 = 0.7283`
- `2014`：WAPE 55.32%，`R^2 = 0.7050`
- `2017`：WAPE 54.61%，`R^2 = 0.6432`

从这些结果可以看出：

- 模型在大多数年份都能把 WAPE 控制在 53%-67% 区间。
- 若从年度总票房规模看，预测值和真实值总体保持同向变化。
- 预测误差并未随着年份单调下降，说明电影市场本身的结构变化和异常影片对误差仍有较大影响。

### 12.3 对回测结果的解释

该结果并不意味着模型已经足以进行商业应用，但说明只使用预算、片长、上映年份、月份周期、语言、国家和类型信息，已经能够解释相当一部分票房变化。这对于课程项目而言，已经展示了较完整的建模与验证思路。

## 13. 多模型比较结果

项目还对四类模型进行了同一回测区间下的比较：

因此，当前项目选择岭回归作为默认模型，是在精度、可解释性与稳定性之间做出的平衡。

## 18. 局限性

尽管项目整体较完整，但仍存在以下局限：

### 18.1 数据时间范围有限

项目只保留到 `2017-07-31`，因此无法反映流媒体时代后期和疫情前后电影市场的变化，也无法描述更近年的发行模式转型。

### 18.2 特征仍偏基础

当前预测模型主要使用结构化元数据，尚未纳入以下重要变量：

- 演员与导演影响力
- 系列 IP 强度
- 营销预算
- 同档期竞争强度
- 奖项口碑与媒体热度
- 发行规模和院线数量

这些因素在真实票房形成机制中都很重要。

### 18.3 小类型样本不稳定

某些类型在特定年份的验证样本非常少，因此会导致：

- 类型级 WAPE 波动很大
- `R^2` 出现极端负值
- 单一年份结果不具备稳定代表性

### 18.4 极端值影响显著

票房本身具有长尾分布特征，少数超级大片会强烈影响：

- MAE
- RMSE
- 百分比误差
- 线性回归斜率

这也是为什么项目需要同时查看 WAPE、`R^2` 和年度总量图，而不能只依赖一个指标。

### 18.5 部分时间序列结论需谨慎解释

极早年份样本量稀少，某些最高评分年份或同比增长年份容易被少量影片放大，因此这些结果更适合作为描述性现象，而不是严肃的产业规律结论。

## 20. 结论

总体而言，本项目成功构建了一个较完整的 IMDB 电影数据分析与票房预测系统。项目以 Kaggle 电影数据为基础，通过统一的数据清洗、探索性分析、高级分析、图表导出、多年滚动回测和桌面界面展示，把原本分散的分析任务整合成了一个可复现、可解释、可展示的完整流程。

从分析结果看，剧情片和喜剧片在样本中占据主导地位；动画、冒险、科幻和家庭类影片在评分或平均票房上表现突出；电影产量在长期内总体增长，并在 2014 年达到样本峰值；预算与票房之间存在显著正相关，但预算并不能单独解释所有市场表现。

从预测结果看，基于预算、片长、上映时间、语言、国家和类型等结构化特征的模型，在 `2003-2017` 的逐年滚动回测中表现出中等水平、但相对稳定的解释力。默认岭回归模型的平均 WAPE 为 `59.98%`，平均 `R^2` 为 `0.6057`，说明该模型能够在一定程度上把握年度票房变化趋势。虽然它距离工业级票房预测仍有明显差距，但作为课程项目，它已经较好地展示了如何把真实数据问题转化为数据工程、描述性分析和预测验证相结合的完整解决方案。

因此，本项目的最大价值并不只是得出若干关于电影市场的统计结论，更在于展示一种完整的数据项目范式：从实际问题出发，通过可追溯的数据清洗、结构化分析、可视化表达和预测回测，把复杂原始数据转化为可解释的洞察和可展示的结果。这使得该项目既具有课程提交价值，也具有作品集展示价值。
